# Naming Conventions

## Resource Groups
rg-<project>-<env>-<region>

Example:
rg-ai-lab-dev-eastus

Purpose: identify the workload  
Environment: dev/test/prod  
Region: Azure region identifier

Benefits:

• easier resource identification  
• consistent resource organization  
• better operational clarity

## App Services
app-<project>-<env>

Example:
app-ai-api-dev

## Key Vaults
kv-<project>-<env>

Example:
kv-ai-secrets-dev

## Storage Accounts
st<project><env>

Example:
staiassetsdev
